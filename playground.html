A cycle the agent can't fake.

code-forge runs three planning phases, then loops a five-phase cycle until the spec is built, then proves it works end to end. Every phase emits a verifiable artifact. Every advancement gate is enforced by a script or a hook, not by agent memory.

  1. Pre-cycle (3). Plan → Spec & e2e → Cycle plan. Codex cross-checks each gate.
  2. Per-cycle (5). Contract → test-list → red → green → review. Loops until cycles are exhausted.
  3. Post-cycle (Phase F). e2e tests prove the whole thing actually works.
