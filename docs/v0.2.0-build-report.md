## Known issues

- **`target_file` semantics in tests.json — present correctness issue.** The schema's example (`"src/sloc.ts"`) names the source file under test; `forge-guard.mjs`'s `checkTestFileEditDuringGreen` blocks edits to those paths during green phase. That is the *opposite* of the rule's stated intent ("block test files during green, anti-weakening"). On a real `/forge` run, the implementer cannot edit the source files it is supposed to write. v0.1.0 shipped and passed smoke because smoke does not exercise the hook against a live cycle; the bug surfaces only on a real `/forge` invocation. **Fix needed in v0.3.x:** split tests.json schema into `target_file` (source under test) and `test_file` (path to block during green); update the hook to consult `test_file`. Until then, the manual /forge dry-run will need a manual workaround (e.g. unset `target_file` in tests.json before green) or the hook needs to be temporarily disabled for that path-class.
- **`inferTargetFromTaskInput` heuristic for `required_subagents`.** The `Task` tool does not expose a target file as a first-class parameter, so the routing rule scans the dispatch's prompt/description for path-shaped tokens. Incidental file paths in prose can trigger false-blocks or miss legitimate routing. The `project_domains` hard rule does not have this problem (it ignores per-target inference). Fix in v0.3.x: extend the hook to read the *contract's* in-scope file list and match each glob against that set, instead of scanning the prompt.
- **Glob fallback YAML parsing is best-effort.** `parseRequiredSubagents` accepts the documented same-line shape (`- match: "..."`); standard YAML where `match:` lives on an indented line under `-` is currently skipped. The smoke fixtures stick to the supported shape; if a user hand-writes a different shape, the binding becomes a silent no-op. Document or upgrade the parser in v0.3.x.
- **No PyYAML in the default environment** — `cycle-validate.sh`'s agent-config.md schema check falls back to presence-only (warns "PyYAML not installed; agent-config.md schema check is presence-only"). The lighter check still catches missing keys; deeper validation is opt-in via `pip install pyyaml`.
- **e2e-extract.sh fallback parser** — handles the documented YAML-in-fenced-block shape; non-trivial YAML (anchors, multi-line scalars) requires PyYAML.
- **Manual /forge dry-run pending** — see "What's NOT yet done." Plan: dry-run on a small TypeScript task, capture artifacts, follow up with a v0.2.1 if the run surfaces issues that the smoke fixtures missed.
