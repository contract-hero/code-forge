# Code Forge v0.4.0 — Scope

A correctness + ergonomics release on top of v0.3.x. Six items carried forward from the v0.3.x build report's known-issues list and the SEDEFI-176 acceptance run's bug-doc amendments.

## Carry-overs

### F8 — Apply F6 pattern to `forge-implementer` (best-of-N coordinator)

**Source:** `docs/BUG-orchestrator-spawned-no-agent-tool.md` Amendment 3 (filed by SEDEFI-176 main session, 2026-04-27 22:00 UTC).

**Problem:** When the main session dispatches `forge-implementer` (the green-phase coordinator), Claude Code's subagent semantics strip the `Agent` tool — the same root cause as the F6 hand-off bug for the orchestrator. The implementer-coordinator was supposed to fan out 6 workers via parallel `Agent` calls; spawned without `Agent` it degrades to best-of-1 and flags the protocol degradation in `synthesis-notes.md`. forge-test produced full best-of-N (6 candidates), but SEDEFI-176's strict-spec session hit it on cycle 1 — meaning the bug is real and reproducible under some session configurations.

**Fix shape (Option A from Amendment 3):**
- Convert `agents/implementer.md` from a dispatchable subagent to a procedure manual (same shape as F6 for the orchestrator).
- Update `skills/code-forge/SKILL.md` "Driving the protocol" section to instruct the main session to dispatch the 6 `implementer-worker`s directly (single parallel-Task message), not via a coordinator.
- The main session reads `agents/implementer.md` inline as bookkeeping guidance and writes `synthesis-notes.md` itself.

### F9 — `state.json` paused/resume schema and resume UX

**Source:** Amendment 2 of the same bug doc.

**Problem:** Failed decision gates (G-Boot, G-Schema, G-Pyth, G-Vault) instruct the orchestrator to "halt and escalate," but the state machine has no formal `paused` field. SEDEFI-176 ad-hoc-added `paused: true`, `paused_at`, `paused_reason`, and a `pause_history` array (richer than the original sketch — worth incorporating).

**Fix shape:**
- Extend `agents/forge-orchestrator.md` state-machine schema with:
  - `paused: bool`
  - `pause_history: [{paused_at, resumed_at?, reason, gate?}]`
- `scripts/forge-status.sh` renders the paused state prominently when present.
- The "driving the protocol" section in SKILL.md adds a resume path: when the main session re-enters `/forge` and sees `paused: true`, it fires `AskUserQuestion` — "Last run paused at <gate> because <reason>. Resume / Abort / Restart?"
- `scripts/cycle-validate.sh` validates the new schema fields when present (still optional for backwards compat with v0.3.x runs).

### F10 — Bash hook coverage

**Source:** v0.3.x review item #3, build-report known-issues, F4's residual gap.

**Problem:** `agents/implementer-worker.md` includes `Bash` in its tool allowlist. forge-guard's PreToolUse rules cover only `Edit/Write/Task/Agent` — a worker that uses `Bash` (`echo > path`, `sed -i`, `cp`, `cat <<EOF >`) to write a test file completely bypasses the F4 prefix-peel check. v0.3.x closed this via prompt discipline only.

**Fix shape:**
- Extend `hooks/hooks.json` matcher to include `Bash`.
- Add `checkBashFileWriteDuringGreen` to `forge-guard.mjs` — parse the Bash command for output-redirect / file-modifying patterns; if the resolved path matches `test_file` or the candidate-staging residue, block.
- Heuristic-only enforcement (parsing arbitrary shell is unsound); accept some false negatives but flag the obvious cases.
- Smoke assertions for `> path`, `>> path`, `cp src test/...`, `sed -i path`.

### F11 — macOS `/var → /private/var` realpath in `makeRepoRelative`

**Source:** v0.3.x review item #5, surfaced via F1 smoke debugging.

**Problem:** `makeRepoRelative(filePath, repoRoot)` does string-prefix comparison. On macOS, `process.cwd()` resolves through `/var → /private/var` symlink, but the supplied `filePath` may not. When the worktree path crosses such a symlink (common in `mktemp`, atypical for production worktrees), `relPath` falls back to the absolute path, which breaks every downstream Set lookup.

**Fix shape:**
- `makeRepoRelative`: call `realpathSync(repoRoot)` and `realpathSync(dirname(filePath))` before string-prefix compare; reconstruct relPath from the resolved parts.
- Smoke fixtures already realpath-resolve via `pwd -P`; remove the workaround once the hook is fixed.

### F12 — `inferTargetFromTaskInput` heuristic too loose for hard-block

**Source:** v0.3.x P8 Codex cross-check; SEDEFI-176 Amendment 1's tangential observation.

**Problem:** `required_subagents` glob fallback infers the dispatch target by scanning `prompt`/`description` for path-shaped tokens. Incidental file paths in prose can mis-trigger the rule (false-positives) or miss legitimate routing (false-negatives). The `project_domains` hard rule doesn't have this issue (it ignores per-target inference, post-F7).

**Fix shape:**
- Replace prompt-scanning with contract-scope-driven matching: read `cycles/N/contract.md`'s `## Files` section to derive in-scope paths; match each glob against that set.
- For pre-cycle dispatches (planner, codebase-explorer), the rule is no-op (no contract yet) — fall back to current behavior or skip.
- Smoke assertions for: contract-derived target match, contract-derived target mismatch, no-contract pre-cycle skip.

### F13 — `parseRequiredSubagents` multi-line YAML

**Source:** v0.3.x P8 Codex cross-check.

**Problem:** The lightweight YAML parser only accepts the documented same-line shape (`- match: "..."`). Standard YAML where `match:` lives on an indented line under a bare `-` is silently skipped — the binding becomes a no-op without warning.

**Fix shape:**
- Either upgrade to a real YAML parser (drop into PyYAML or a tiny Node YAML library) for `agent-config.md`, OR extend `parseRequiredSubagents` to accept the multi-line shape.
- Smoke fixtures cover both shapes.

## Out of scope (deferred further)

- Diversity injection in best-of-N (different prompt seeds per worker) — spec §4.6 v0.3.x deferral, still v0.5.x material.
- `dispatch-in-pane.sh` (fresh CLI sessions for verdict gates) — spec §15 deferral.
- Re-running the full bench against v0.3.x and v0.4.x — done as a separate exercise, not a release artifact.

## Sequencing

F8 is the highest-priority fix because it's the same architectural class as F6 and currently degrades best-of-N silently. F9 is UX polish but heavily user-facing. F10/F11/F12/F13 are correctness fixes.

Suggested order: **F11 → F12 → F13 → F8 → F9 → F10**.
- F11 unblocks F8's smoke fixtures (mktemp paths cross symlinks).
- F12/F13 are independent code fixes.
- F8 lands the architectural change; F9 layers the resume UX on top.
- F10 last because Bash hook coverage interacts with everything and benefits from the rest landing first.

- Codex cross-check after F8 lands (architectural change).
